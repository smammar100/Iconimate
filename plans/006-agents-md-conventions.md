## Why this matters

`CLAUDE.md` is just `@AGENTS.md`, and `AGENTS.md` is a generic "this is a modified Next.js, read
`node_modules/next/dist/docs/` before writing code" stub. Neither documents the load-bearing repo
contracts. Since this project is actively developed by agents (the icon-authoring workflow is
agent-driven), an agent editing an icon or adding a motion token will silently break the registry
generator with no guidance that these are tripwires:

- The generator (`scripts/build-registry.mjs`) **regex-parses** `registry/icons/index.ts` and
  `components/dark/icon-meta.ts`; reformatting an entry (multi-line import, comment inside the entry
  object, reordered fields) silently degrades or throws.
- `parseSource` (`:136`) only recognizes a **closed set** of import modules (`react`, `motion/react`,
  `@/lib/motion-tokens`, `@/lib/icon`, `@/hooks/use-hover`, `./_*` factories) — any other import in a
  registry icon throws `unexpected import module`.
- Adding an export to `registry/lib/motion-tokens.ts` requires updating the generator's token
  dependency map (`TOKEN_DEPS`) or the transitive-closure resolution breaks.
- Every icon's animated **rest state must equal the original Phosphor glyph** (the pixel-diff
  fidelity rule used throughout development) — undocumented anywhere an agent would find it.

## Steps

### Step 1: Author the AGENTS.md content

Keep the existing "read the bundled Next.js docs" note, and add these sections (write them concisely,
each pointing at the authoritative file rather than duplicating it):

1. **Repo shape** — one paragraph: animated-icon gallery (`app/`, `components/dark/`) + shadcn
   registry (`registry/`), generated by `scripts/build-registry.mjs` into `public/r/` and
   `registry/*.gen.*`. Build/verify commands (`pnpm dev`, `pnpm build`, and — once plan 001 lands —
   `pnpm verify`).

2. **Icon anatomy** — the required shape of a `registry/icons/<slug>.tsx`: `"use client"`,
   `forwardRef<IconHandle, IconProps>`, `useHover()`, `useImperativeHandle` exposing
   `startAnimation`/`stopAnimation`, `normal` + `animate` variants, a reduced-motion static fallback.
   Point at one real icon as the exemplar. State the **rest-state fidelity rule**: the `normal`
   variant must render pixel-identically to the original Phosphor glyph.

3. **The generator contract (tripwires)** — list the three hand-maintained source-of-truth files
   (`registry/icons/index.ts`, `components/dark/icon-meta.ts`, plus the icon `.tsx` itself) that must
   be edited in lockstep to add an icon, and warn that `index.ts`/`icon-meta.ts` are **regex-parsed**
   — do not reformat entries (keep the `{ slug, name, keywords }` field order; single-line imports).
   State the closed import allow-list for registry icons (`react`, `motion/react`,
   `@/lib/motion-tokens`, `@/lib/icon`, `@/hooks/use-hover`, `./_*` factories) — any other import
   breaks the build. Note that adding a `motion-tokens.ts` export requires updating `TOKEN_DEPS` in
   the generator. Cite `scripts/build-registry.mjs:1-12` and `:164-170` as the authority.

4. **Lab workflow** — prototype variants live in `app/lab/<slug>/page.tsx`; promote a chosen variant
   into `registry/icons/<slug>.tsx` + `index.ts` + `icon-meta.ts`.

5. **A "before you commit" checklist** — run `pnpm verify` (or `pnpm build`), keep the icon's rest
   state pixel-identical, don't reformat the regex-parsed files.

**Verify**: the doc names all three source-of-truth files, the import allow-list, the `TOKEN_DEPS`
requirement, and the rest-state rule.
