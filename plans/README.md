## Dependency notes

- **001 is the keystone.** It creates `pnpm verify` (typecheck + generator golden tests + lint), the
  safety net every risky plan (008/009/010/011/012) uses to prove it didn't break the generator or
  the type-check. Land it first.
- **008 → 009.** Both touch the no-flash theme mechanism. Remove S2 first (theme sync becomes pure
  CSS `color-scheme`), *then* replace the server `cookies()` read with an inline pre-hydration
  script so the layout can prerender statically. Reversing the order risks FOUC regressions.
- **007 → 013.** 013 (shareable `?q=` search + canonical per-icon routes) builds on the `?icon=`
  reader from 007. 013 is a design/spike plan, not build-everything.
- **002, 003, 004, 005, 006, 007** are independent P1/P2 quick wins; they can run in any order,
  ideally before the riskier perf trio so 001's gate protects the latter.
