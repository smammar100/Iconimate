## Why this matters

Three different, hard-coded icon counts exist, none matching reality (174 registered icons):

- `README.md:10` says `` `30 icons` `` and `:85` "Thirty icons and counting".
- `lib/seo.ts:22` exports `ICON_COUNT = 147`, which is woven into the JSON-LD description, every
  `KEY_FACTS` sentence, the FAQ answers, the `/llms.txt` AI-crawler summary, the docs page, and the
  Lucide comparison page.

These are the exact surfaces the project built to sell its breadth to search engines and AI
assistants. The stale `147` (and the 6×-wrong README `30`) actively undersell and misrepresent the
library. Fixing them is the cheapest credibility win available, and deriving the number kills future
drift.

## Steps

### Step 1: Derive `ICON_COUNT` from the generated metadata

In `lib/seo.ts`, replace `export const ICON_COUNT = 147;` with a value computed from the generated
visible-icon array:

```ts
import { visibleIconMeta } from "@/registry/icon-meta.gen";
export const ICON_COUNT = visibleIconMeta.length;
```

Confirm `@/registry/icon-meta.gen` is import-safe from `lib/seo.ts` (it is a plain data module — no
"use client", no side effects). If `lib/seo.ts` is imported by both server and client components,
this is fine: `icon-meta.gen.ts` is pure data.

**Verify**: `pnpm build:registry && pnpm exec tsc --noEmit` → exit 0; `node -e "import('@/lib/seo')"`
style check or a temporary log confirms `ICON_COUNT` equals `visibleIconMeta.length`.
